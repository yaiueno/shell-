\subsubsection*{問題2}
目的:\par
シェル変数の代入と参照、単引用符・二重引用符、コマンドライン引数、算術展開の違いを確認する。

コード:\par
\begin{lstlisting}
#!/bin/sh
ABC=A; A=BC
echo A$ABC
exit 0

#!/bin/sh
echo '$1' + '$2' = $(($1 + $2))
echo "$1" + "$2" = $((1 + 2))
exit 0
\end{lstlisting}
出力例:\par
\begin{lstlisting}
$ ./1.sh
AA
$ ./2.sh 8 12
$1 + $2 = 20
8 + 12 = 3
\end{lstlisting}
方法・考察:\par
シェル変数の代入では、等号の前後に空白を入れない必要がある。単引用符で囲んだ \texttt{\$1} や \texttt{\$2} は展開されず文字列として表示され、二重引用符で囲んだ場合は引数の値に展開される。

\begin{lstlisting}

\texttt{test} で引数が10未満かを判定し、10未満なら引数を1増やして同じスクリプトを再帰的に実行する。10以上になった時点で値を表示して終了する。

\subsubsection*{問題10}

\begin{lstlisting}
#! /bin/sh
case "$2" in
  "+") echo $(($1 + $3)) ;;
  "-") echo $(($1 - $3)) ;;
  "*") echo $(($1 * $3)) ;;
  "/") echo $(($1 / $3)) ;;
  "%") echo $(($1 % $3)) ;;
esac
\end{lstlisting}
出力例:\par
\begin{lstlisting}
$ ./calc.sh 8 + 12

$ ./calc.sh 8 '*' 12
96
$ ./calc.sh 8 / 3
2
\end{lstlisting}
方法・考察:\par
\texttt{case} 文で第2引数の演算子を判定し、対応する算術演算を実行した。\texttt{*} はシェルのワイルドカードとして解釈されるため、実行時には引用符で囲む必要がある。

\subsection*{4.2 実習2}

目的:\par
実習1の問題2で扱ったシェル変数、引用符、引数、算術展開について、実際のコードと実行結果をまとめる。特に、単引用符と二重引用符による変数展開の違い、および \texttt{\$((...))} による整数演算の方法を理解する。

\subsubsection*{スクリプト}
\begin{lstlisting}
#!/bin/sh
ABC=A; A=BC
echo A$ABC
exit 0

#!/bin/sh
echo '$1' + '$2' = $(($1 + $2))
echo "$1" + "$2" = $((1 + 2))
exit 0
\end{lstlisting}

\subsubsection*{実行例}
\begin{lstlisting}
$ ./1.sh
AA
$ ./2.sh 8 12
$1 + $2 = 20
8 + 12 = 3
\end{lstlisting}

\subsubsection*{結果・考察}
代入文では等号の前後に空白を入れない。\texttt{ABC=A; A=BC} とした場合、\texttt{ABC} には \texttt{A} が入るため、\texttt{echo A\$ABC} の結果は \texttt{AA} になる。また、単引用符内の \texttt{\$1} は展開されず、二重引用符内の \texttt{\$1} は引数の値に展開される。\par

\subsection*{4.3 実習3}

目的:\par
\texttt{x=abc}, \texttt{y=1+2}, \texttt{z=-2} を設定し、複数の条件式を \texttt{if} 文に入れて、どの条件が真になるかを確認する。\texttt{test} コマンドや \texttt{[ ]} による文字列比較、数値比較、否定、複合条件の書き方を理解する。

\subsubsection*{スクリプト}
\begin{lstlisting}
#!/bin/sh
x=abc; y=1+2; z=-2
if (*) ; then
    echo "true"
fi
exit 0
\end{lstlisting}

\subsubsection*{確認した条件式}
\begin{lstlisting}
(1) test x$x = "xabc"
(2) [ $xx = "abcx" ]
(3) test $y -eq 3
(4) test 1+2 = y
(5) [ test $z = -2 ]
(6) [ -$z = 2 ]
\end{lstlisting}

\subsubsection*{実行例}
\begin{lstlisting}
(1) 真
(2) 偽
(3) エラー（1+2 は数値として扱えない）
(4) 偽
(5) エラー（[ test ... ] は不正な条件式）
(6) 偽
\end{lstlisting}

\subsubsection*{結果・考察}
\texttt{test} や \texttt{[ ]} では、文字列比較は真偽が判定できる一方で、数値比較に文字列を渡すとエラーになる。また、\texttt{[ test ... ]} のように条件式を重ねて書くと構文として不正になり、正しく評価されないことが分かった。\par

\texttt{x=abc} のときは \texttt{x\$x} が \texttt{xabc} となるため (1) は真になった。\texttt{y=1+2} は整数ではないので (3) は数値比較に失敗し、\texttt{z=-2} を使った条件でも書き方が不適切なものはエラーまたは偽になる。条件式は、変数展開後の形まで含めて正しく記述する必要がある。\par

\subsubsection*{問題 21}

\subsubsection*{問題 22}
目的:\par
\texttt{sed} の正規表現と後方参照を使い、文字列の一部を取り出したり置換したりする。

方法:\par
\texttt{echo} で用意した文字列を \texttt{sed} に渡し、パターンに一致する部分を取り出したり置換したりした。

スクリプト/コマンド:\par
\begin{lstlisting}
echo "<html><body>" | sed 's/<\([^>]*\)>.*/\1/'
echo "int main (void)" | sed 's:(\([a-z]*\)):\1:'
echo "int main (void)" | sed 's:.*(\([a-z]*\)):\1:'
echo "/tmp/mytest.sh.tgz" | sed 's:[^.]*\(\..*\):\1:'
echo "/tmp/mytest.sh.tgz" | sed 's=[/.]=-=g'
\end{lstlisting}

実行例:\par
\begin{lstlisting}
$ echo "<html><body>" | sed 's/<\([^>]*\)>.*/\1/'
html
$ echo "int main (void)" | sed 's:(\([a-z]*\)):\1:'
int main void
$ echo "int main (void)" | sed 's:.*(\([a-z]*\)):\1:'
void
$ echo "/tmp/mytest.sh.tgz" | sed 's:[^.]*\(\..*\):\1:'
.sh.tgz
$ echo "/tmp/mytest.sh.tgz" | sed 's=[/.]=-=g'
-tmp-mytest-sh-tgz
\end{lstlisting}

考察:\par
\texttt{\(\)} で囲んだ部分は \texttt{\textbackslash 1} により再利用できる。\texttt{!} を区切り文字にした場合は、対話シェルが履歴展開として解釈してエラーになることがあったため、ここでは \texttt{:} を区切り文字として用いた。

\vspace{0.7cm}

\subsubsection*{問題 23}
目的:\par
複数の \texttt{bar*.tex} ファイルに含まれる誤記 \texttt{bgein} を \texttt{begin} に一括置換する。

方法:\par
\texttt{for} 文で対象ファイルを順に処理し、\texttt{sed} の結果を一時ファイルに出力してから元のファイルへ戻す。

スクリプト/コマンド:\par
\begin{lstlisting}
#!/bin/sh
for x in ./bar*
do
    sed 's=bgein=begin=g' "$x" > tmp.tex
    mv tmp.tex "$x"
done
echo "finish"
\end{lstlisting}

実行例:\par
\begin{lstlisting}
$ ./23/23.sh
finish
\end{lstlisting}

考察:\par
\texttt{sed} の結果を直接同じファイルへ書き戻すことはできないため、一時ファイルを経由して置換結果を保存した。実際に確認するには、対象となる \texttt{bar*.tex} を用意してから実行する必要がある。

\subsection*{4.6 実習6}

目的:\par
正の整数の入力 \texttt{n} に対し、\texttt{n} 以下の正の奇数の和を求めるシェルスクリプト \texttt{j6.sh} を作成する。\texttt{while} 文と算術展開を用いて、条件を満たす数値だけを順に加算する方法を確認する。

方法:\par
\texttt{i} を1から始め、\texttt{i} が入力値以下である間、\texttt{sum} に \texttt{i} を加える。その後 \texttt{i} を2増やすことで、正の奇数だけを順に処理する。

スクリプト:\par
\begin{lstlisting}
#!/bin/sh
i=1
sum=0

while [ $i -le $1 ]
do
    sum=$((sum + i))
    i=$((i + 2))
done

echo $sum
exit 0
\end{lstlisting}

実行例:\par
\begin{lstlisting}
$ ./j6.sh 5
9
$ ./j6.sh 10
25
\end{lstlisting}

考察:\par
\texttt{i} を2ずつ増やすことで、1, 3, 5, ... の正の奇数だけを取り出せる。入力値が偶数の場合は、その値を超えない最大の奇数までが加算対象になる。\par

\section*{全体の考察}

シェルスクリプトは、PythonやJavaなどの汎用プログラミング言語と比較して、
実行環境への依存度が低い。インタプリタのインストールやランタイムの準備が
不要であり、POSIX準拠のシェルさえあればほぼどのUNIX系OSでも動作する。
この特性から、OSのセットアップ直後・システム復旧時・組み込み環境など、
高水準な言語処理系が利用できない低次元な場面での自動化処理に適している。

一方で、複雑なデータ構造の操作や数値演算、可読性の高いコードが求められる
場合には、PythonやC言語などの汎用言語の方が適切である。

したがって、シェルスクリプトはファイル操作・プロセス管理・環境設定など
OSに密接した処理に用い、ロジックが複雑になる場合は他言語と組み合わせる
という使い分けが実用上は効果的である。
